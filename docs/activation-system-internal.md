# 激活码系统 — 内部技术文档

> 本文档供开发和运营参考，请勿对外公开。

---

## 一、系统概述

激活码系统用于管理 AI 工具的授权分发。管理员在后台为每个工具生成激活码，发给付费用户，用户在用户中心输入激活码后解锁工具下载权限。

---

## 二、激活码格式

```
XXXX-XXXX-XXXX-XXXX
```

- **长度**：16 个有效字符，分4组，用 `-` 分隔
- **字符集**（31个字符）：

  ```
  A B C D E F G H J K L M N P Q R S T U V W X Y Z 2 3 4 5 6 7 8 9
  ```

- **排除的字符**：`0`（易与 O 混淆）、`1`（易与 I/L 混淆）
- **生成方式**：Node.js `crypto.randomInt()`，密码学安全随机数
- **总组合数**：31^16 ≈ 7.9 × 10²³（约 79 bit 熵值）

---

## 三、安全性

| 项目 | 说明 |
|------|------|
| 随机性 | 使用 `crypto.randomInt()`，非 `Math.random()`，不可预测 |
| 唯一性 | 生成时与数据库已有码去重，code 字段设唯一索引 |
| 不可推算 | 无序列号、无日期编码、无规律，纯随机 |
| 单次使用 | 每个码仅能激活一次，用后状态变为 `used` |
| 身份验证 | 激活必须登录，激活记录绑定 userId |
| 暴力破解 | 按每秒1000次请求计算，穷举约需 25万亿年 |

## 四、激活码状态流转

```
生成 → unused（未使用）
         ↓ 管理员分配
       assigned（已分配）
         ↓ 用户输入激活
       used（已使用）

任意状态 → expired（已作废，管理员操作）
expired  → unused（恢复，管理员操作）
```

| 状态 | 含义 | 可激活 |
|------|------|--------|
| `unused` | 刚生成，未分配 | 是 |
| `assigned` | 已分配给指定用户 | 仅该用户可用 |
| `used` | 已被使用 | 否 |
| `expired` | 已作废 | 否 |

## 五、数据库结构

```sql
CREATE TABLE "ActivationCode" (
    "id"        TEXT PRIMARY KEY,
    "code"      TEXT NOT NULL UNIQUE,    -- 激活码，如 A7KN-3PBR-WDXY-8HM4
    "toolId"    TEXT NOT NULL,           -- 对应工具 ID
    "userId"    TEXT,                    -- 分配/使用的用户 ID
    "status"    TEXT DEFAULT 'unused',   -- unused | assigned | used | expired
    "note"      TEXT,                    -- 管理员备注
    "createdAt" DATETIME,
    "usedAt"    DATETIME
);
```

索引：`code`（唯一）、`toolId`、`userId`、`status`

## 六、API 接口

### 管理员接口（需 admin 权限）

| 方法 | 路径 | 说明 |
|------|------|------|
| GET | `/api/admin/tools/{toolId}/codes` | 获取激活码列表，支持 `?status=` 过滤 |
| POST | `/api/admin/tools/{toolId}/codes` | 批量生成激活码 |
| PUT | `/api/admin/tools/{toolId}/codes` | 更新激活码（分配/作废/恢复） |

**POST 参数：**
```json
{
  "count": 10,        // 生成数量，1~100
  "note": "发给XX客户"  // 可选备注
}
```

**PUT 参数：**
```json
{
  "codeId": "xxx",
  "action": "assign" | "expire" | "reset",
  "userId": "xxx",    // assign 时必填
  "note": "备注"       // 可选
}
```

### 用户接口（需登录）

| 方法 | 路径 | 说明 |
|------|------|------|
| POST | `/api/tools/activate` | 激活码激活 `{ "code": "XXXX-XXXX-XXXX-XXXX" }` |
| GET | `/api/tools/activate` | 获取当前用户已激活的工具列表 |

## 七、后台操作指南

### 设置工具下载地址

1. 进入 `/admin/tools`
2. 点击工具的编辑按钮
3. 在「下载地址」字段填入百度网盘链接（含提取码）
4. 保存

### 生成激活码

1. 在工具列表中，点击工具行右侧的🔑按钮
2. 设置生成数量（建议按需生成，不要一次太多）
3. 可选填备注，如「2026年2月-张三」
4. 点击「生成激活码」
5. 点击「复制全部未使用」一键复制，发给客户

### 查看和管理

- 通过状态筛选查看未使用/已使用的激活码
- 可以作废泄露的激活码
- 可以恢复误作废的激活码

---

## 八、运营建议

1. **按需生成**：不要一次生成太多激活码，建议每批 5~20 个
2. **善用备注**：记录每批激活码发给了谁，方便追踪
3. **定期清理**：作废长期未使用的激活码
4. **保密发放**：激活码通过微信私聊等私密渠道发给用户，不要公开发布
5. **提取码分离**：百度网盘链接和提取码可以分开发送，增加安全性
